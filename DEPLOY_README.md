# 오늘부터

> 실패해도 괜찮아요, 오늘부터 다시 — 간헐적 단식 루틴 형성 앱
> 제작: AYoom

이 리포지토리는 Claude와 함께 설계·개발한 "오늘부터" 앱의 실제 배포용 프로젝트입니다.
자세한 기능 명세는 함께 전달된 `PRD_v2.md`를 참고하세요.

---

## 1. 로컬에서 실행해보기

```bash
npm install
npm run dev
```

터미널에 뜨는 주소(보통 `http://localhost:5173`)를 브라우저에서 열면 바로 확인할 수 있어요.

## 2. 프로덕션 빌드

```bash
npm run build
```

`dist/` 폴더에 정적 파일이 생성됩니다. 이 폴더를 어떤 정적 호스팅에든 그대로 올리면 동작해요.

로컬에서 빌드 결과를 미리 보고 싶다면:

```bash
npm run preview
```

---

## 3. Vercel로 배포하기 (추천, 가장 쉬움)

**방법 A — 웹에서 바로 (Git 저장소가 있을 때)**
1. 이 폴더를 GitHub 저장소로 올려요.
2. [vercel.com](https://vercel.com) → "Add New Project" → 그 저장소 선택
3. Framework Preset은 **Vite**로 자동 인식돼요. Build Command는 `npm run build`, Output Directory는 `dist` — 보통 자동으로 채워져요.
4. Deploy 누르면 끝. 몇 분 안에 `https://프로젝트명.vercel.app` 주소가 생겨요.

**방법 B — CLI로 (Git 없이 폴더만 있을 때)**
```bash
npm install -g vercel
vercel
```
질문에 답하면(프로젝트명 등) 그대로 배포돼요. 이후 `vercel --prod`로 정식 배포할 수 있어요.

## 4. Netlify로 배포하기 (대안)

1. [netlify.com](https://netlify.com) → "Add new site" → "Deploy manually"
2. `npm run build`로 만든 `dist` 폴더를 그대로 드래그 앤 드롭
3. 몇 초 안에 배포 완료, 랜덤 주소가 생성돼요 (나중에 원하는 이름으로 바꿀 수 있어요)

## 5. 카카오톡/문자로 지인에게 테스트 배포하기

위 방법으로 배포한 URL을 그대로 공유하면 돼요. 별도 앱스토어 심사나 설치가 필요 없는 웹앱이라, 링크만 열면 바로 써볼 수 있어요. (iOS/Android 모두 브라우저에서 정상 동작해요.)

## 6. 알아두어야 할 것 (현재 버전의 한계)

PRD에 P0/P1로 명시된 기능은 모두 구현되어 있지만, 아래 항목은 아직 그렇지 않아요:

| 항목 | 현재 상태 |
|---|---|
| 데이터 저장 위치 | **이 기기(브라우저)의 localStorage에만 저장**돼요. 다른 기기·다른 브라우저에서는 기록이 안 보여요. 브라우저 데이터를 지우면(캐시 삭제 등) 기록도 함께 사라져요. |
| 로그인 / 서버 동기화 | 아직 없어요 (PRD상 P2, 계획된 기능). 설정 화면에 "준비 중"으로 자리만 있어요. |
| 변화 기록(몸사진) | 하단 탭·기록 허브까지는 만들어져 있지만, 실제 촬영·실루엣 가이드 기능은 "곧 추가돼요" 안내만 있는 상태예요. |
| 알림(푸시) | 앱 안에서 배너로만 안내되고, 실제 브라우저/OS 푸시 알림은 연동돼 있지 않아요. |
| 엑셀 리포트의 차트 | 무료 엑셀 라이브러리 한계로 실제 도형 차트 대신 텍스트 막대/스파크라인으로 대체돼 있어요. (자세한 내용은 대화 기록 참고) |
| 다국어 | 한국어만 지원해요. |

## 7. 기술 스택

- React 18 + Vite
- Tailwind CSS
- [lucide-react](https://lucide.dev) — 아이콘
- [recharts](https://recharts.org) — 체중 변화 그래프
- [xlsx (SheetJS)](https://sheetjs.com) — 엑셀 건강 리포트 내보내기
- 상태 저장: 브라우저 `localStorage` (서버 없음, 완전 프론트엔드 단독 동작)

## 8. 다음에 이어서 개발할 때 참고할 것

- 데이터 구조(`defaultData()` 함수 참고)는 이미 실제 서버 DB로 이관하기 쉽게 설계돼 있어요. 로그인/서버 동기화를 붙일 때는 `loadData`/`saveData` 두 함수만 API 호출로 교체하면 큰 구조 변경 없이 이어갈 수 있어요.
- 이번 빌드는 체중 진행률 그래프 등 이미지(base64)가 소스에 일부 포함되어 최종 번들 용량이 다소 커요(약 1MB). 실 서비스 단계에서는 이미지들을 `public/` 폴더의 별도 파일로 분리하는 걸 추천해요.
